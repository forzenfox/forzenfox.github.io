Spring 层次性上下文事件传播

  • 发生说明

    当 Spring 应用出现多层次 Spring 应用上下文（ApplicationContext）时，如 Spring WebMVC、SpringBoot 或 Spring Cloud 场景下，由子 ApplicationContext 发起 Spring 事件可能会传递到其 Parent ApplicationContext（直到 Root）的过程

  • 如何避免

    • 定位 Spring 事件源（ApplicationContext）进行过滤处理
 1public class HierarchicalSpringEventPropagateDemo {
 2
 3    public static void main(String[] args) {
 4
 5        AnnotationConfigApplicationContext parentContext = new AnnotationConfigApplicationContext();
 6        parentContext.setId("parent-context");
 7
 8        AnnotationConfigApplicationContext currentContext = new AnnotationConfigApplicationContext();
 9        currentContext.setId("current-context");
10
11        // 设置层次性
12        currentContext.setParent(parentContext);
13
14        //注册监听器
15        parentContext.register(MyListener.class);
16        currentContext.register(MyListener.class);
17        parentContext.register(MyListener2.class);
18        currentContext.register(MyListener2.class);
19
20        parentContext.refresh();
21        currentContext.refresh();
22
23        // 关闭顺序会影响输出结果：
24        // 先close parentContext的话，close currentContext的时候 parentContext 中的监听器已经被移除了，无法监听到事件
25        currentContext.close();
26        parentContext.close();
27    }
28
29}

33    @Override
34    public void onApplicationEvent(ApplicationContextEvent event) {
35        System.out.printf("MyListener [%s] 监听到事件 %s\n",
36                event.getApplicationContext().getId(),
37                event.getClass().getSimpleName());

42class MyListener2 implements ApplicationListener<ApplicationContextEvent> {

同步和异步 Spring 事件广播

  • 基于实现类 - org.springframework.context.event.SimpleApplicationEventMulticaster

    • 模式切换：setTaskExecutor(java.util.concurrent.Executor) 方法
      • 默认模式：同步
      • 异步模式：如 java.util.concurrent.ThreadPoolExecutor
    • 设计缺陷：非基于接口契约编程

  • 基于注解 - @org.springframework.context.event.EventListener

    • 模式切换
      • 默认模式：同步
      • 异步模式：标注 @org.springframework.scheduling.annotation.Async
    • 实现限制：无法直接实现同步/异步动态切换

  • 源码分析：SimpleApplicationEventMulticaster#multicastEvent

     1public void multicastEvent(final ApplicationEvent event, @Nullable ResolvableType eventType) {
 2    ResolvableType type = (eventType != null ? eventType : resolveDefaultEventType(event));
 3    Executor executor = getTaskExecutor();
 4    for (ApplicationListener<?> listener : getApplicationListeners(event, type)) {
 5        if (executor != null) {
 6            // executor 默认是 null，所以默认是同步执行
 7            executor.execute(() -> invokeListener(listener, event));
 8        }
 9        else {
10            invokeListener(listener, event);
11        }
12    }
13}

  • 代码示例：

     1@EnableAsync
 2public class AsyncEventHandlerDemo {
 3
 4    public static void main(String[] args) {
 5        AnnotationConfigApplicationContext context = new AnnotationConfigApplicationContext();
 6
 7        // @EventListener 无法生效
 8        // context.addApplicationListener(new CustomizedSpringEventListener());
 9
10        // @EnableAsync 生效
11        context.register(AsyncEventHandlerDemo.class);
12        context.register(CustomizedSpringEventListener.class);
13
14        context.refresh();
15
16        ApplicationEventMulticaster applicationEventMulticaster =
17                context.getBean(AbstractApplicationContext.APPLICATION_EVENT_MULTICASTER_BEAN_NAME,
18                        ApplicationEventMulticaster.class);
19
20        if (applicationEventMulticaster instanceof SimpleApplicationEventMulticaster) {
21            SimpleApplicationEventMulticaster simpleApplicationEventMulticaster =
22                    (SimpleApplicationEventMulticaster) applicationEventMulticaster;
23
24            // 切换 taskExecutor
25            ExecutorService taskExecutor = newSingleThreadExecutor(new CustomizableThreadFactory("my-spring-event-thread-pool-"));
26            // 同步 -> 异步 (全局切换)
27            simpleApplicationEventMulticaster.setTaskExecutor(taskExecutor);
28
29            // 通过监听 上下文关闭事件 关闭线程池
30            applicationEventMulticaster.addApplicationListener(new ApplicationListener<ContextClosedEvent>() {
31                @Override
32                public void onApplicationEvent(ContextClosedEvent event) {
33                    if (taskExecutor.isShutdown()) {
34                        return;
35                    }
36                    taskExecutor.shutdown();
37                }
38            });
39        }
40
41        context.publishEvent(new CustomizedSpringEvent("异步事件"));
42
43        context.close();
44    }
45}

Spring Boot 和 Spring Cloud 上的应用

Spring Boot 事件

事件类型 发生时机
ApplicationStartingEvent 当 Spring Boot 应用已启动时
ApplicationStartedEvent 当 Spring Boot 应用已启动时
ApplicationEnvironmentPreparedEvent 当 Spring Boot Environment 实例已准备时
ApplicationPreparedEvent 当 Spring Boot 应用预备时
ApplicationReadyEvent 当 Spring Boot 应用完全可用时

事件类型 发生时机

InstanceRegisteredEvent 当服务实例注册后
RefreshEvent 当 RefreshEndpoint 被调用时
