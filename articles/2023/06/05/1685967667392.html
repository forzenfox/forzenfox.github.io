基础架构图

Server 层

  • MySQL 大多数核心服务功能
  • 所有的内置函数
  • 所有跨存储引擎的功能 （存储过程、触发器、视图等）
连接器

TCP连接池 使用长连接模式复用 TCP 连接

  • 建立、维持、管理连接
  • 获取权限
1mysql -h$ip -P$port -u$user -p

用户密码通过，连接器会到权限表查询登录用户拥有的权限，并用于后续的权限逻辑判断

意味着当前连接中的权限是连接建立那一刻的权限，即使对当前用户的权限进行了修改，已建立的连接权限依旧不会更新，新建的连接才会获取最新的权限设置。

查询缓存

不同客户端之间共享，查询缓存对频繁更新的表意义不大。

设置默认不使用查询缓存

1query_cache_type = DEMAND

然后在 SQL 语句中显示指定使用查询缓存

1select SQL_CACHE * from tb_test where id=10;

MySQL 8.0 之后，查询缓存被删除

存储引擎层

存储引擎层负责数据的存储和提取。

1create table tb_test ( id int ) engine=memory;
