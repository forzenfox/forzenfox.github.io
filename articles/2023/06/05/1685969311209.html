Spring Environment 抽象

  • 统一的 Spring 配置属性管理

    Spring Framework 3.1 开始引入 Environment 抽象，它统一 Spring 配置属性的存储，包括占位符处理和类型转换，不仅完整地替换 PropertyPlaceholderConfigurer，而且还支持更丰富的配置属性源（PropertySource）

  • 条件化 Spring Bean 装配管理

    通过 Environment Profiles 信息，帮助 Spring 容器提供条件化地装配 Bean

Spring Environment 接口使用场景

  • ⽤于属性占位符处理
  • 用于转换 Spring 配置属性类型
  • 用于存储 Spring 配置属性源（PropertySource）
  • 用于 Profiles 状态的维护

条件配置 Spring Profiles

Spring 3.1 条件配置

  • API：org.springframework.core.env.ConfigurableEnvironment

    默认实现：org.springframework.core.env.StandardEnvironment

    • 获取配置

      • getActiveProfiles()

        返回为此环境显式激活的配置文件集。配置文件用于创建要按条件注册的 bean 定义的逻辑分组，例如基于部署环境。可以通过将“spring.profiles.active”设置为系统属性或调用 ConfigurableEnvironment.setActiveProfiles(String...)来激活配置文件。
        如果没有将配置文件明确指定为活动，则任何默认配置文件都将自动激活。

      • getDefaultProfiles()

        当没有明确设置活动配置文件时，默认情况下将配置文件集返回为活动状态。

    • 修改配置

      • addActiveProfile(String)

        将配置文件添加到当前的活动配置文件集中。

      • setActiveProfiles(String...)

        指定为此 Environment 活动的配置文件集。在容器引导期间评估配置文件以确定是否应向容器注册 bean 定义。
        任何现有的活动配置文件都将替换为给定的参数；使用零参数调用以清除当前的活动配置文件集。使用 addActiveProfile 添加配置文件，同时保留现有集

      • setDefaultProfiles(String...)

        如果没有其他配置文件通过 setActiveProfiles 显式激活，则指定默认激活的配置文件集。

    • 匹配配置

      • acceptsProfiles(String...) 5.1 之后废弃

        返回一个或多个给定配置文件是否处于活动状态，或者在没有明确的活动配置文件的情况下，返回一个或多个给定配置文件是否包含在一组默认配置文件中。如果配置文件以“！”开头逻辑是相反的，即如果给定的配置文件不活动，该方法将返回 true 。例如， env.acceptsProfiles("p1", "!p2")如果配置文件 'p1' 处于活动状态或 'p2' 不活动，将返回 true 。
        已弃用
        从 5.1 开始支持接受 acceptsProfiles(Profiles)
        抛出：
        IllegalArgumentException – 如果使用零参数调用或任何配置文件为 null 、空或仅空格

      • acceptsProfiles(Profiles)

        返回活动配置文件是否与给定的 Profiles 谓词匹配

  • 注解：@org.springframework.context.annotation.Profile

基于 API 扩展 Spring 配置属性源

  • Spring 应用上下文启动前装配 PropertySource

     1MutablePropertySources propertySources = context.getEnvironment().getPropertySources();
 2
 3// 动态地插入 PropertySource 到 PropertySources 中
 4Map<String, Object> source = new HashMap<>();
 5source.put("user.name", "dylan");
 6MapPropertySource propertySource = new MapPropertySource("first-property-source", source);
 7propertySources.addFirst(propertySource);
 8
 9// 启动 Spring 应用上下文
10context.refresh();

  • Spring 应用上下文启动后装配 PropertySource

    1// 启动 Spring 应用上下文
2context.refresh();
3
4// 上下文启动后 修改配置
5source.put("user.name", "007");

样例代码：

 1public class EnvironmentPropertySourceChangeDemo {
 2
 3    @Value("${user.name}")
 4    private String userName;
 5
 6    public static void main(String[] args) {
 7        AnnotationConfigApplicationContext context = new AnnotationConfigApplicationContext();
 8        // 注册 Configuration Class
 9        context.register(EnvironmentPropertySourceChangeDemo.class);
10
11        MutablePropertySources propertySources = context.getEnvironment().getPropertySources();
12
13        // 动态地插入 PropertySource 到 PropertySources 中
14        Map<String, Object> source = new HashMap<>();
15        source.put("user.name", "dylan");
16        MapPropertySource propertySource = new MapPropertySource("first-property-source", source);
17        propertySources.addFirst(propertySource);
18
19        // 启动 Spring 应用上下文
20        context.refresh();
21
22        // 上下文启动后 修改配置
23        source.put("user.name", "007");
24
25        EnvironmentPropertySourceChangeDemo environmentPropertySourceChangeDemo = context.getBean(EnvironmentPropertySourceChangeDemo.class);
26
27        System.out.println(environmentPropertySourceChangeDemo.userName);
28
29        for (PropertySource ps : context.getEnvironment().getPropertySources()) {
30            System.out.printf("PropertySource(name=%s) 'user.name' 属性：%s\n", ps.getName(), ps.getProperty("user.name"));
31        }
32
33        // 关闭 Spring 应用上下文
34        context.close();
35    }
36}
